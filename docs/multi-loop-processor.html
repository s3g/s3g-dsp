Multi Loop Processor

s3g Multi Loop Processor 8ch is a CLAP instrument plugin for combining up to four loaded audio files into one eight-lane loop instrument.

It shares the stable loop engine model with Loop Processor: lane playheads, loop region controls, seam crossfade, lane relationship controls, and an eight-channel output bus. The difference is the source layer. Multi Loop builds an internal eight-channel composite from several files before the loop engine plays it.

Workflow

  • Insert s3g Multi Loop Processor 8ch on an 8-channel REAPER track.
  • Click LOAD and select up to four files in the file picker.
  • Choose a RULE to decide how files feed the eight lanes.
  • Use SRAT and XBLD to shape the source layer.
  • Use the shared loop engine controls to set rate, loop region, seam crossfade, and output gain.
  • Optionally set MIDI to GATE or TRIG so note input can launch and transpose playback.
  • Use SYNC to realign lane playheads after editing loop timing.

When a lower-channel file is loaded, its channels repeat across the eight lanes. Stereo maps as odd/even, quad repeats channels 1-4, and 8-channel files map directly. Files wider than eight channels are spread across the source channel range so the eight lanes sample the wider file rather than only reading channels 1-8.

Source Display

The SOURCES panel shows four source slots. Each loaded source has a compact waveform drawn from the most energetic channel in that file. The row also shows the file channel count, sample rate, and displayed channel number.

The larger waveform below the source slots shows the generated eight-lane composite. Loop markers and lane playheads are drawn on that composite view.

Source Rules

Rule Meaning
ORDERGroups lanes by loaded-file order. Useful when each file should occupy a clear lane region.
INTERInterleaves sources across lanes. This is the default and often gives the most immediate multichannel spread.
RNDUses a deterministic shuffled assignment. It sounds random but recalls consistently with the same loaded sources.
MORPHBlends from one source to the next across the eight lanes. Use XBLD to make the transitions hard or smooth.

Source Controls

Control Meaning
RULEChooses the source-to-lane assignment rule.
SRATSource rate spread. Each loaded file receives a deterministic rate offset before the loop engine plays the composite.
XBLDSource crossblend. In MORPH, low values make sharper source zones and high values make smoother interpolation.

SRAT and XBLD rebuild the source composite after the control is released. This keeps the audio thread stable while still allowing audible source-layer changes.

Loop Engine

The remaining controls match the single-file Loop Processor model.

Control Meaning
RATEBase playback rate for all lanes after the source composite is built.
XFDLoop seam crossfade as a percentage of the active loop length.
DUCKOptional seam ducking inside the crossfade window.
STRTLoop start position inside the composite.
LENLoop length. The default is the full composite.
OUTOutput gain after lane playback.
MIDIMIDI input behavior. OFF ignores notes, GATE plays while notes are held, and TRIG retriggers from note-on.
ROOTRoot MIDI note for pitch-follow playback rate. Note 60 is middle C.

Relationships

Relationship controls shape per-lane timing after the source composite is built. This lets one source rule produce several different motion feels without changing the loaded files.

Control Meaning
SPRDBipolar lane-rate spread.
DRFTBipolar slow drift amount.
CTRRelationship center for the lane spread.
GLDGlide time for relationship changes.

Standalone App

The macOS standalone app uses the same shared DSP engine, but runs outside REAPER with AVAudioEngine and CoreMIDI. Its MIDI map follows the nanoKONTROL2 control layout.

Sliders CC0-CC7 control rate, spread, drift, center, glide, start, length, and output. Knobs CC16 and CC17 control crossfade and ducking. SOLO buttons CC32-CC35 choose the source rule. Transport buttons control play, stop, sync, rule advance, rebuild, and toggle play. MIDI notes are ignored in the standalone app.

If the active audio device has fewer than eight outputs, the app folds the eight internal lanes down across the available output channels for auditioning.

Practical Notes

Multi Loop Processor is separate from topology plugins. It is a loaded-audio instrument with source rules and lane relationships, not a point-field processor.

For a simpler single-file workflow, use Loop Processor. Use Multi Loop when several loops, textures, or multichannel files should be arranged into one eight-lane playable instrument.

Session Checks

For multichannel sessions, place a meter after the plugin and try combinations such as mono plus stereo, stereo plus quad, stereo plus 8-channel, and stereo plus 24-channel files. The eight output lanes should match the selected source rule while loop-region, crossfade, ducking, rate, spread, drift, and center changes remain smooth.
