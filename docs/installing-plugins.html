Installing Pre-release Plugins

These notes are for installing a packaged macOS CLAP zip from a GitHub pre-release. Plugin names, parameters, saved states, and included plugins may change.

The supported development target is macOS + REAPER.

Requirements

  • A macOS pre-release zip from the s3g-dsp GitHub releases page.
  • REAPER.

Download

Pre-release builds are attached to GitHub releases when available. If no zip is available, use the Building From Source page.

Install

Unzip the pre-release package and copy the included .clap bundles to the user CLAP plugin folder:

~/Library/Audio/Plug-Ins/CLAP/

Replace older bundles with the same names if prompted. REAPER may need a plugin rescan after installation.

Current Plugins

The current source tree builds effects and instruments separately. Effects process incoming track audio. Instruments generate sound from internal or MIDI excitation.

topology delay s3g Delay Processor 8ch / 24ch Main topology-driven delay processor and wider 3OAFX insert build. fold-down s3g MC to Stereo Autogain 128-channel input to true stereo output fold-down prototype. 3OAFX boundary s3g 3OAFX Send Decoder / Return Encoder Work-in-progress boundary pair for 3OAFX insert chains.

Instruments

in development s3g Resonant Terrain 8ch 8-channel resonant terrain instrument with topology-distributed resonators and internal/MIDI excitation.

REAPER Setup

After installing, open REAPER and rescan CLAP plugins if the new builds do not appear automatically. Search for s3g in the FX browser.

For multichannel testing, set the track channel count before inserting the plugin.

For s3g MC to Stereo Autogain, open REAPER's plugin pin connector and enable Zero out unmapped output channels. The CLAP plugin has a true stereo output bus, but REAPER can still pass unmapped track channels around the effect unless this option is enabled.

For the larger REAPER package workflow, see the s3g-mc 3OAFX Send Return Controller guide.

Cleanup

If older pre-release test plugins appear in REAPER, remove stale bundles from the user CLAP folder and rescan:

~/Library/Audio/Plug-Ins/CLAP/

Build folders can also contain stale artifacts after plugin renames. Reconfigure and rebuild with the current preset if needed.
