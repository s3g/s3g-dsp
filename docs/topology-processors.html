Topology Effects

Topology effects turn selected audio lanes into points and map the relationships among those points to effect parameters. They suit multichannel material whose internal structure should move as one network rather than as unrelated channels.

Start with the Framework

Topology Framework explains the shared point, patch, camera, neighbor, centroid, and automation model before each effect gives that network a different DSP meaning.

Plugins

  • Delay Processor 8ch/24ch: maps point distance and direction to delay time, feedback, filtering, modulation, and level.
  • Wave Geometry Processor 8ch: maps the network to fold, clip, sample-step, bit-collapse, scope, and tape-head behavior without an FFT.
  • Spectral Topology Processor 8ch/24ch: maps topology to FFT-bin processing and displays the result as a grayscale sonogram.

Shared Model

A lane patch selects the participating channels. Those lanes become points, and motion changes their distances, clusters, and directional relationships. Each effect interprets the same network differently, so automation can preserve a recognizable spatial or structural gesture while changing the DSP model.

Delay Processor is the best first effect example because its display makes the mapping especially direct.

Routing

Choose the fixed-width build that matches the REAPER track. These are general multichannel effects and do not assume ACN/SN3D; use Ambisonic Effects when channels form an encoded sound field.
