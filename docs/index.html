s3g-dsp

s3g-dsp is a BSD-3-Clause C++/CLAP plugin project for native multichannel DSP experiments that support the larger s3g-mc REAPER package.

The current focus is reliable multichannel lane processing in REAPER, beginning with the s3g Delay Processor. It uses a topology layer where active channels become points, points form a neighbor network, and the network maps to per-lane DSP values without exposing every lane as a host automation parameter.

Topology Layer

The topology layer is the common control idea for future s3g-dsp plugins. A patch matrix selects active lanes. Those lanes become points in a compact 3D field. Shape, displacement, animation, and variants move the points. The plugin then derives per-lane DSP values from point position, radius, lane phase, nearest neighbors, and centroid pull.

Build and Install

The local development build uses the CLAP preset and installs bundles into the macOS CLAP plugin folder for REAPER testing.

cmake --preset clap
cmake --build --preset clap
./scripts/install-clap-bundles.sh
