Spectral Topology Processor

s3g Spectral Topology Processor 8ch/24ch is a topology-shaped spectral processor for fixed-width multichannel tracks. It uses the shared topology model to vary spectral spray, freeze, damage, repeat, and phase behavior across active lanes.

The processor analyzes short overlapping FFT frames, then moves, holds, repeats, removes, or blurs their frequency bins before resynthesis. Use it for spectral motion that remains coordinated across lanes; choose Wave Geometry for direct waveform shaping or a Macro effect for a smaller shared control set.

Signal Flow

  1. REAPER audio enters the internal patch matrix.
  2. Active outputs become topology lanes.
  3. The topology map derives lane-specific spectral parameters.
  4. The processed output can be viewed in the SONO page as a grayscale post-processing sonogram.

Engine Controls

  • BINS: spectral spray range.
  • DRFT: bin drift and movement across the spectral field.
  • HOLD and FRZ: hold and freeze behavior for spectral material.
  • FDBK: bounded spectral feedback.
  • SMR: spectral smear.
  • HOLE: dropout cells in the spectral path.
  • PHAS: phase blur.
  • BRK and RPT: broken-bin and spectral-repeat layers.
  • TILT: frequency tilt across the processor.
  • MIX: dry/wet mix.
  • OUT: final output gain.

Topology

The topology panel uses the same control language as the Delay Processor and Wave Geometry Processor. Point position, radius, neighbor relationships, and centroid movement shape how the spectral controls differ from lane to lane.

The topology window has two pages:

  • TOPO: point network plus topology heatmap view.
  • SONO: one-second, grayscale post-processing sonogram for the lane outputs.

The sonogram is a visual analyzer for the plugin output. It is not an additional audio effect, and it does not change the spectral processing.

Patch Matrix

The patch matrix selects which REAPER inputs feed the spectral lanes. Use it when the track is wider than the source material or when one input should feed several lanes. The 24-channel build uses a compact matrix so all lanes remain visible.

REAPER Notes

Use the 8-channel build for eight-lane material and the 24-channel build for wider fixed-width workflows. In either case, route the REAPER FX pins first, then use the plugin matrix for lane-level decisions.
