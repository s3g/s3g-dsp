Layout Panner

s3g Layout Panner is a direct spatial panner: 64 source inputs are distributed across a fixed 64-channel speaker output bus. It does not encode to ambisonics internally. It is useful when the speaker layout itself is the target.

The plugin is grouped with 3OAFX tools because it shares the same spatial layout language, camera controls, and custom speaker-design workflow as the ambisonic plugins. It can also be used independently for multichannel panning in REAPER.

input16 sources
output64 speakers
routingdirect gain
layoutpreset/custom

Signal Shape

source inputs 1-64
-> source AED / gain / mute / solo
-> selected speaker layout
-> direct source-to-speaker gain render
-> speaker outputs 1-64

Only the active speaker count writes sound. Unused output lanes are silent. Source mute and solo are available for diagnosis and performance setup.

Pages

Page Use
FIELD Shows speaker geometry and source points together. Drag source points in the current camera plane for free spatial positioning.
MIXER Shows source gain faders, mute, solo, and output level without crowding the spatial field.
DESIGN Edits the speaker layout. Source points are hidden so speaker selection, dragging, and JSON import/export stay clear.

Layout Presets

The layout menu provides common speaker shapes such as quad, rings, cube layouts, dodeca, domes, and custom. Presets are starting points. Switch to CUSTOM or edit a speaker to build a layout for a specific room or format.

Speaker markers use the same square AED color strategy as the Ambi Point Encoder and Speaker Decoder. Geometry lines show the intended layout mesh, not nearest-neighbor analysis.

Panner Variants

The direct panner family uses the same source mixer, speaker layout language, and 64-channel output convention, but offers different gain laws for different musical and calibration needs.

Plugin Method Use
s3g Layout Panner distance / cosine General direct panning with a design page for custom layouts.
s3g DBAP Panner distance-based amplitude panning Smooth distance-weighted distribution across the selected layout.
s3g LBAP Panner layer/local-lobe amplitude panning Topology-aware lobe panning that follows layout tiers and local speaker neighborhoods.
s3g VBAP Panner vector-base amplitude panning Sharper vector panning against the selected layout topology.

LBAP and VBAP use an internal solver topology derived from the speaker layout. The GUI overlay draws the actual topology used by the solver: faint cyan for LBAP and faint yellow for VBAP. Imaginary solver speakers may be added where the layout has weak or incomplete topology. These points help the solver but do not add output channels.

For flat 2D layouts, LBAP and VBAP add silent imaginary top and bottom speakers. Moving source elevation away from the speaker plane therefore becomes a smooth level change rather than a meaningless control. For partial 3D layouts, such as quad plus overheads, imaginary support points and pole-cap blending stabilize motion near +90 and -90 degrees elevation.

Design Page

DESIGN is for speaker geometry only. It hides panning method, focus, diffusion, and other audio-distribution controls so the page stays about the speaker layout.

  • SHAPE chooses the custom layout generator: AUTO, RING, DOME, GEO, or STACK.
  • COUNT sets active speakers from 2 to 64.
  • AUTO chooses useful count-based defaults. For example, 8 speakers generates a cube.
  • SEL, AZ, EL, and DST edit one speaker at a time.
  • COPY copies the current preset into a custom editable layout.
  • LOAD and SAVE import/export readable JSON layout files.

Custom Layout JSON

Saved layouts use readable JSON with a shape label, speaker count, and one AED entry per speaker.

{
  "shape": "AUTO",
  "speaker_count": 8,
  "speakers": [
    { "azimuth": -45.0, "elevation": -35.0, "distance": 1.0 }
  ]
}

Imported files preserve their speaker coordinates. The shape label is metadata after import, so loading a measured layout does not regenerate over the edited positions.

REAPER Notes

Use a track with enough channels for the desired speaker output. The plugin exposes a stable 64-channel output bus so REAPER pin routing remains predictable as layouts change.

The plugin is not a replacement for the Ambi Point Encoder or Speaker Decoder. Use those when you need an ACN/SN3D ambisonic signal path. Use Layout Panner when you want direct source-to-speaker panning.
