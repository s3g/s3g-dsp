Spectral Processors

This group collects newer native C++ CLAP processors that grew out of spectral and Gen/RNBO prototyping. The production path in s3g-dsp is hand-built C++ DSP; RNBO-generated source is not part of this repository.

2-in / 16-out Cascade Taps Stepped tap-ring processor with base/step timing, decay, damping, direction, and a SOFT control that widens handoff windows and reins in risky hot cascades.

Click And Feedback Conditioning

These processors intentionally reach into unstable-sounding territory, so the wrappers and DSP include guardrails for finite output, peak bounds, parameter smoothing, transport-stop de-clicks, and impulse stress cases. The goal is not to remove percussive behavior; it is to make sharp or feedback-heavy settings less likely to produce accidental clicks or runaway levels.

Cascade Taps exposes this most directly with SOFT. Higher values widen internal handoff windows, raise effective minimum timing for short cascades, and reduce risky wet/decay combinations. Lower values keep the stepped, percussive character more exposed.

Routing

Spectral Spray is a stereo effect. Shard Scatter, Orbit Delay, and Cascade Taps are two-input, sixteen-output effects, so the REAPER track channel count must be high enough for their output bus. Use REAPER's pin connector when inserting them into an existing multichannel chain.

Validation

The local smoke test exercises the spectral FFT path, spectral spray output, high-feedback spectral spray behavior, automation smoothing, shard sparse and stop cases, orbit stop de-clicks, cascade impulse suppression, and multichannel output bounds.
