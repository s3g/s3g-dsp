Ambi Ray Encoder

s3g Ambi Ray Encoder places a mono source and listener inside a sampled room field and produces a 64-channel ACN/SN3D output bus. Motion of either point changes the direct path and early reflection directions and delays; source motion also interpolates the sampled late-room profile.

Workflow

  1. Open s3g-mc Ray Sketch and design, generate, or import a space.
  2. Place the listener and default source, then choose the source-sampling grid and reflection-slot count.
  3. Export a .s3gray Ray Field.
  4. Insert the plugin on a 64-channel REAPER track, route a mono source to its input, and load the field.
  5. Set the ambisonic order, move the source and listener in the top and side views, and balance direct, early, and late sound.
  6. Follow the encoder with an ambisonic decoder or meter.

Ray Field

The built-in room makes the plugin immediately usable. LOAD accepts version 1 .s3gray files with the format identifier s3g-ambi-ray-field. A field stores room and navigation bounds, a reference listener and default source position, and 1 to 256 sampled source cells. Each cell can hold as many as 32 directional early reflections plus a late response profile. See the .s3gray format reference for the complete runtime contract.

At runtime, the encoder blends the four nearest cells as the source moves. Loaded field data is stored with the REAPER project state, so the original .s3gray file does not need to remain beside the project for recall.

Musical Interpretation

Ray Sketch resolves the designed space into a compact map of stable reflection slots and late-room profiles. The plugin owns both live positions. Its direct path is calculated continuously between source and listener, nearby source cells reshape the early and late field, and bounce-aware early paths are recalculated through their wall, floor, ceiling, portal, or echo point as either endpoint moves. Moving either point therefore does not require rebuilding or streaming a new ray trace.

The result is intentionally a musical room instrument rather than an architectural simulator. Material curves, path losses, and late estimates preserve a space's directional character and repeatable motion behavior without claiming measurement-grade acoustics.

Position and Motion

SOURCE X/Y/Z and LISTENER X/Y/Z are normalized automation parameters mapped to the loaded field's navigation bounds. The GUI reports both positions in metres. MOVEMENT applies the same smoothing time to both points.

The display places a top view and side view next to each other. Select SRC or LIS in the Position panel, or grab either marker directly, then drag in the top view for X/Y or the side view for X/Z. Solid green lines run from source through an exported amber bounce point to the listener. Dashed cyan events without a defensible bounce point use a stable world-space arrival anchor derived from the field's reference listener. The grey source-to-listener line is the direct path. Position automation is continuous; loading a new field constructs a replacement room runtime outside the audio callback.

The late field remains a directional room bed interpolated from source cells. Listener motion reshapes direct and early geometry but does not claim a second sampled late-field dimension.

Room Controls

  • DIRECT sets the distance-delayed source component.
  • EARLY sets the interpolated directional reflection field.
  • LATE sets the diffuse late response.
  • SIZE scales propagation and room timing.
  • SCATTER increases reflection variation and diffusion.
  • WIDTH scales the directional spread of the room response.
  • AIR adds distance-dependent high-frequency damping.
  • BYPASS ROOM removes early and late energy while retaining direct spatial encoding.

Order and Routing

The direct source can use first through seventh order. The room response is represented through third order, and channels above the selected order are cleared. Decode the output with Ambi Speaker Decoder, Ambi Stereo Decoder, or Ambi Head Decoder.
