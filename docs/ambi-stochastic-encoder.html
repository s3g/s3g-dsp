Ambi Stochastic Encoder

s3g Ambi Stochastic Encoder 64 is a 64-voice stochastic synthesis instrument with first-through-seventh-order ACN/SN3D output. It can run freely, respond to MIDI, or combine both behaviors.

Each voice owns four evolving pressure-curve generators. Their breakpoint amplitudes and segment durations are controlled by reflected second-order random walks: one walk changes the step, and that step changes the audible curve. Segment durations set oscillator period directly, so pitch can leave its starting region and continue evolving.

Workflow

  1. Insert s3g Ambi Stochastic Encoder 64 on a 64-channel REAPER track.
  2. Choose a factory preset or begin from FREE PERIOD.
  3. Choose FREE, MIDI, or BOTH.
  4. Set the ambisonic ORDER and active voice count.
  5. Choose a generator selection law and transition method.
  6. Shape the random walks, time-fields, topology, and AED projection.
  7. Follow the instrument with an ambisonic decoder or meter.

User Presets

SAVE writes the current state as a portable .s3gstochastic file. LOAD restores one through the macOS file chooser. The file contains every synthesis, selection, time-field, envelope, topology, projection, ambisonic-order, and output setting.

User presets are versioned JSON documents and can be moved between systems or kept with a project archive. Changing a control after loading or selecting a preset marks the current state as CUSTOM. REAPER project state continues to store the complete plugin state independently.

PTSSets 4 to 32 breakpoints per generator.

The distribution menus provide UNIFORM, GAUSS, CAUCHY, LOGISTIC, ARCSINE, EXPON, and BINARY. The amplitude and timing walks can use different distributions.

Controls that become turbulent near their upper limit use a tapered GUI response. The lower and middle regions receive more slider travel, while the most extreme walk, density, spread, and topology values remain available near the right edge. Displayed values, user-preset values, and REAPER automation remain unchanged.

BASE, SPREAD, and DEV seed the initial periods. They are not pitch centers after initialization. LOW reflects the total polygon period before it becomes subaudio while preserving the relative duration of every segment. Generator identity and topology Z distribute voices upward from that floor. MIDI notes reseed a voice when assigned.

Generator Selection

A selection law chooses one of each voice's four generators at a waveform boundary. A choice persists across several cycles before the law is sampled again, giving each evolving pressure curve time to become audible. The history strip shows recent choices.

LawBehavior
RANDOMChooses any generator independently.
SERIESUses every generator once before reshuffling.
WEIGHTWeights generator choice from the voice's X position.
TENDENCYMoves a probability center through the generator bank.
MARKOVUses the previous choice and topology direction to weight the next one.
WALKUses another reflected second-order walk to move through generator indices.

XFER lets nearby voices exchange generator tendencies and walk motion, and sets how strongly a new event can propagate to its nearest neighbors. MEM controls selection-law memory, generator persistence, and the stability of the slower macro state.

LINK moves to the selected generator, MERGE interlaces it with the previous generator, and VARY creates a topology-weighted variation between them. A short interpolation at each cycle boundary keeps the transition continuous without blurring the pressure curve for the full cycle.

Time-Fields

Free-running voices use renewal clocks: every active event is followed by a genuine rest before that voice can begin again. DENS sets the target activity, DUR sets the mean event duration, CONT increases duration variation and clustered propagation, and REST sets the minimum inactive time. The ADSR envelope smooths each change of state.

DENS is compensated for the active voice count: the same setting uses a lower per-voice target as the field grows from 8 toward 64 voices. Gentle activity regulation steers the ensemble toward that target without hard voice limits. This keeps the texture articulated instead of automatically becoming continuous; even at maximum density, REST remains in force.

Each onset can schedule delayed events at the two nearest topology points. Distance determines propagation time, while XFER and CONT set cascade strength. The network lines brighten as a cascade arrives, making clustered activity visible as well as audible.

Macro Scenes

The autonomous engine also moves through metastable macro scenes. A scene slowly changes four internal relationships together: activity, event duration, pressure-curve mutation, and cascade strength. MACRO sets the approximate time spent in each region. Higher MEM makes the changes smaller and more persistent; lower values allow wider movement between contrasting states.

In MIDI mode, note gates replace the autonomous time-fields. BOTH allows either source to open a voice.

Topology and Projection

The main display is both a synthesis map and the source of each voice's AED placement. Points move in one persistent topology; there is no separate physics or spatial-motion layer.

  • X biases pressure-curve mutation and generator weighting.
  • Y changes event probability, field duration, and selection tendency.
  • Z biases the segment-duration walk and therefore oscillator period.
  • Distance from the center increases walk drive and transition variation.
  • Nearest-neighbor links carry generator influence, walk motion, and delayed event cascades between voices.

SHAPE, ANIM, RATE, AMT, DEPTH, SCALE, COLL, and TWIST reshape and animate the shared field. AZIM, ELEV, and DIST place its center. FOLLOW sets how fully topology position and radius are projected into AED space.

Click a point to inspect its current and next generators. Drag blank field space to change the camera; use TOP, SIDE, 3/4, and the zoom buttons to restore useful views.

Pressure Display

The gray curve is the selected voice's current generator and the brighter curve is the next generator. Breakpoint markers belong to the next curve. A1 and T1 report the current primary-walk barriers for amplitude and duration.
