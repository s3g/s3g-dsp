Ambi Grain Processor

s3g Ambi Grain Processor is a CLAP instrument plugin for loaded ACN/SN3D ambisonic media. It creates grain events in source time and writes those grains into a circular multichannel tape buffer, so every encoded channel follows the same grain timing.

formatACN/SN3D
output16ch 3OA
typeCLAP instrument
stateprototype

Workflow

  • Insert s3g Ambi Grain Processor on a 16-channel REAPER track.
  • Click LOAD and choose an ambisonic audio file.
  • Use ORD to choose whether the loaded file is treated as 1OA, 2OA, or 3OA.
  • Follow the plugin with an ambisonic decoder, stereo decoder, head decoder, or energy visualizer.

The output bus is fixed at 16 channels. First-order and second-order settings fill the active ambisonic channels and leave higher-order channels silent.

Grain Model

The processor uses one shared event stream. A grain has one source position, duration, rate, direction, and window. That event reads every active ambisonic channel at the matching sample time, applies a Parzen-style grain envelope, and accumulates the result into a circular 16-channel tape buffer before playback.

This follows the older Max Gen patch model more closely than direct live grain summing. The tape layer lets overlapping grains build a field while preserving encoded channel relationships. This is different from independent per-channel granulation, which can smear or destabilize the spatial image.

Controls

Control Meaning
ORDAmbisonic order to process: 1OA, 2OA, or 3OA.
MODESource-time mode: scan, cloud, freeze, or stepped jump.
DENSGrain density in events per second. Long grain sizes reduce the available density range for stable playback.
GMSGrain size in milliseconds, from short cuts to long 4-second sparse cloud windows.
S-SRCScan source. At 0%, SCAN reads through the full file; above 0%, it scans from the chosen source point.
J-SRCJump source used by JUMP mode.
JITSource-position jitter around the current grain position.
RATEGrain playback rate.
RJITPer-grain rate jitter in octave range.
REVChance that a grain reads backward.
F-SRCFreeze source position used by FREEZE mode.
J-STPNumber of source-position steps used by JUMP mode.
SYNCRegular grain spacing or randomized asynchronous spacing around the same density.
ENVGrain window: Parzen, sine, Hann, triangle, or Gaussian.
OUTOutput gain after density compensation.

Testing Notes

This is a fresh prototype rebuilt from the Max Gen/HOA granular direction and the current CLAP package framework, not a restored committed plugin. The first test target is simple: load known 1OA, 2OA, or 3OA media, confirm the following decoder receives coherent motion, and check that mode and density changes do not produce runaway levels or broken channel order.

If REAPER routing behaves strangely after prior unstable projects, test in a new REAPER project first. That keeps host-state issues separate from plugin behavior.
