3OAFX Displacement

s3g 3OAFX Displacement is a 16-channel third-order ACN/SN3D insert. It loads geometry timelines exported by the s3g-mc Displacement Score browser utility and applies them to an encoded ambisonic field.

Signal Flow

3OA input (16 channels)
-> matched decode to the fixed 24-point 3OAFX sphere
-> score-driven point displacement and distance processing
-> matched dynamic encode
-> 3OA output (16 channels)

The matched encoder makes the source geometry an identity transform. An amount of 0%, or the score's source scene, preserves the incoming field apart from the Output control. Geometry and distance coefficients ramp across each host block to avoid vector-boundary steps.

Score Files

Use LOAD SCORE to open a version 1 s3g-mc-displacement-score JSON file. The plugin reads up to 64 scenes, uses shortest-path azimuth interpolation, and embeds the parsed score in project state. The browser utility remains the authoring surface; the plugin is a playback and interpretation surface.

The player reads the raw radius stored in scene targets so its distance mode remains useful even when the browser preview was exported with radius normalization enabled.

Controls

Free mode advances independently at the selected Rate and continues through Loop or Palindrome playback. Sync mode derives position directly from the host's absolute beat timeline, so playback, seeking, and project reopening return to the same score position instead of depending on an accumulated internal phase.

Distance Interpretation

GAIN treats radius as a capped inverse-distance level law. Radius 1.0 is unity, nearer points can rise to at most +6 dB, and farther points can fall to at most -12 dB.

PHYSICAL keeps that safe gain law and adds relative propagation delay for farther points, distance-dependent air damping, and gradual high-order reduction as points recede. Nearer points are not given negative delay: doing that would require fixed lookahead and host latency. This keeps neutral geometry at zero added latency while still making outward displacement read as physical depth.

REAPER Routing

Put the plugin on a 16-channel 3OA track after an encoder or other 3OA process and before an ambisonic decoder. Set the REAPER track to at least 16 channels and verify the plugin pin connector remains mapped one-to-one.
