Ambi Path Encoder

s3g Ambi Path Encoder 64 encodes up to 64 source lanes into first-through-seventh-order ACN/SN3D ambisonics while each source follows a path trajectory.

Use it when spatial motion should be authored as a reusable shape or gesture rather than as only point automation or generative cloud motion.

Model

  • INPUTS sets the active source lanes.
  • ORDER selects 1OA through 7OA.
  • PATHS sets the number of active trajectories.
  • ASSIGN maps sources to one path, round-robin paths, or source-indexed paths. Source-indexed mapping wraps across the active path count when there are more inputs than paths.
  • PLAY switches between stopped, running, and phase-scrubbed playback.

Trajectory

The plugin field can define paths directly. Use EDIT to add, delete, or move path points. In TOP view, click empty field space to add a path point and drag handles to edit horizontal position. In SIDE view, click or drag handles to edit height against the side projection. DEL PT or the Delete key removes the selected point, and CLEAR empties the selected path so the next click starts a new drawing from scratch. Use PLAY when the field should act as a camera surface: click and drag in any view to inspect the path without editing it.

  • RAND generates editable paths for the active PATHS count.
  • DEV sets how far generated paths deviate from an orderly orbit.
  • PTS sets how many points each generated path receives.
  • MODE selects one-shot, loop, or palindrome playback.
  • INTERP selects linear, Catmull-Rom spline, or hold interpolation.
  • SYNC switches between free-running internal motion and host-transport beat sync.
  • DIV sets the beat length of one path cycle when sync is enabled.
  • SCRUB in the PLAY menu freezes automatic motion and lets PHASE directly scan through the path. Automate PHASE when the path should behave like a drawable automation lane.
  • RATE, PHASE, and SPREAD control free-running speed and source offsets. The plugin GUI gives the low end of RATE extra slider travel for slow path motion.
  • SMOOTH slews source positions after path sampling.
  • EASE bends path phase toward smoother in/out motion.
  • DIST scales the path radius before encoding.

Files

JSON is the canonical path format for saving and recalling encoder trajectories. SVG is an interchange format for importing or exporting geometric path drawings.

  • LOAD JSON reads s3g_ambi_path files containing one or more paths.
  • SAVE JSON writes the active paths with 3D point coordinates.
  • IMPORT SVG reads a practical subset of external SVG geometry: path, polyline, and polygon coordinate data.
  • EXPORT SVG writes a simple 2D projection of the active paths for editing or documentation.

SVG import treats the drawing as trajectory geometry, not as artwork. Stroke color, fills, effects, and layer styling are ignored.

Routing

Place the plugin on a wide REAPER track carrying source lanes. Decode the encoded output with Ambi Speaker Decoder, Ambi Stereo Decoder, or Ambi Head Decoder.
