Building From Source

These notes are for building the current s3g-dsp CLAP plugins from source. This is pre-release software: plugin names, parameters, saved states, and included plugins may change.

The supported development target is macOS + REAPER.

Requirements

  • CMake 3.20 or newer.
  • A C++17 compiler, such as Apple Clang through Xcode Command Line Tools.
  • macOS for the current CLAP GUI/plugin bundles.
  • REAPER.
  • Optional: clap-validator for plugin validation.

Smoke Test

The development preset builds the DSP smoke test without CLAP bundles:

cmake --preset dev
cmake --build --preset dev
./build/s3g_dsp_smoke
./build/s3g_3oafx_displacement_smoke
./build/s3g_ambi_imprint_safety_smoke
./build/s3g_ambi_ray_encoder_smoke
./build/s3g_psd_raw_field_smoke
./build/s3g_psd_raw_field_parameter_audit

Build CLAP Plugins

Build the CLAP plugins:

cmake --preset clap
cmake --build --preset clap

By default, the build fetches CLAP headers with CMake. To use an existing local CLAP checkout, configure manually with S3G_FETCH_CLAP=OFF and S3G_CLAP_INCLUDE_DIR.

cmake -S . -B build-clap \
  -DS3G_BUILD_CLAP_PLUGIN=ON \
  -DS3G_FETCH_CLAP=OFF \
  -DS3G_CLAP_INCLUDE_DIR=/path/to/clap/include
cmake --build build-clap

WORLD speech vocoder support is enabled by default for Ambi Vox Encoder and fetched at a pinned revision. Disable it only when a build without Ambi Vox WORLD analysis is required:

cmake -S . -B build-clap \
  -DS3G_BUILD_CLAP_PLUGIN=ON \
  -DS3G_ENABLE_WORLD=OFF

Build Standalone Apps

macOS standalone app targets are built separately from the CLAP plugin bundle targets:

cmake --preset apps
cmake --build --preset apps

The standalone app targets are:

build-apps/apps/multi_loop_processor_standalone/s3g Multi Loop Processor.app
build-apps/apps/vox_builder/s3g Vox Builder.app

Vox Builder uses the same pinned WORLD dependency as Ambi Vox Encoder. Its segmentation core can be checked independently with ./build-apps/s3g_vox_builder_smoke.

Install Local Build

Install the CLAP bundles:

./scripts/install-clap-bundles.sh

The default install location is:

~/Library/Audio/Plug-Ins/CLAP/

The install script copies bundles from build-clap/plugins/ and replaces matching older bundles.

Validate

Check the static documentation and advisory GUI conventions:

python3 scripts/check-docs.py
./scripts/audit-gui-style.sh

If clap-validator is installed, validate one or more built or installed bundles. For example:

clap-validator validate --only-failed \
  build-clap/plugins/clap_macro_shred/s3g_macro_shred_mono.clap \
  build-clap/plugins/clap_macro_shred/s3g_macro_shred.clap \
  build-clap/plugins/clap_macro_shred/s3g_24ch_macro_shred.clap \
  build-clap/plugins/clap_psd_raw_field/s3g_fault.clap

Package a Pre-release

After a complete CLAP build, create the macOS zip used for GitHub pre-releases:

./scripts/package-macos-clap-prerelease.sh

The archive is written to dist/ and includes 79 CLAP bundles, VOT wavetables, the Ambi Vox demo voicebank, the project license, and third-party notices. The script ad-hoc signs and strictly verifies every bundle by default; set S3G_CODESIGN_IDENTITY to use another macOS signing identity. Set S3G_RELEASE_VERSION and S3G_RELEASE_DATE to override the pre-release metadata. Apple notarization remains a separate release step.

Cleanup

If old plugin names remain after a rename, reconfigure and rebuild with the current preset.
