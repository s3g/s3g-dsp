Building From Source

These notes are for compiling the current s3g-dsp CLAP plugins from source. The repository is still pre-release, so plugin names, parameter mappings, state formats, and the included plugin set may change.

The current development target is macOS with REAPER as the tested host. Other operating systems and DAWs are not expected support targets at this stage.

statepre-release
formatCLAP
platformmacOS
hostREAPER

Requirements

  • CMake 3.20 or newer.
  • A C++17 compiler, such as Apple Clang through Xcode Command Line Tools.
  • macOS for the current native CLAP GUI/plugin bundle targets.
  • REAPER for the current tested host workflow.
  • Optional: clap-validator for plugin validation.

Smoke Test

The development preset builds the shared DSP smoke test without building CLAP bundles:

cmake --preset dev
cmake --build --preset dev
./build/s3g_dsp_smoke

Build CLAP Plugins

Build the CLAP plugins with the saved CMake preset:

cmake --preset clap
cmake --build --preset clap

By default, the build fetches CLAP headers with CMake. To use an existing local CLAP checkout, configure manually with S3G_FETCH_CLAP=OFF and S3G_CLAP_INCLUDE_DIR.

cmake -S . -B build-clap \
  -DS3G_BUILD_CLAP_PLUGIN=ON \
  -DS3G_FETCH_CLAP=OFF \
  -DS3G_CLAP_INCLUDE_DIR=/path/to/clap/include
cmake --build build-clap

Install Local Build

Install the built CLAP bundles into the user CLAP plugin folder:

./scripts/install-clap-bundles.sh

The default install location is:

~/Library/Audio/Plug-Ins/CLAP/

The install script currently copies the active plugin bundles from build-clap/plugins/ and replaces existing bundles with the same names.

Validate

If clap-validator is installed, validate installed bundles with commands like:

clap-validator validate \
  ~/Library/Audio/Plug-Ins/CLAP/s3g_8ch_delay_processor.clap --only-failed
clap-validator validate \
  ~/Library/Audio/Plug-Ins/CLAP/s3g_24ch_delay_processor.clap --only-failed
clap-validator validate \
  ~/Library/Audio/Plug-Ins/CLAP/s3g_24ch_passthrough_test.clap --only-failed

Cleanup

Build folders can contain stale generated artifacts after plugin renames or target culls. Reconfiguring and rebuilding with the current preset should only recreate the active targets.
