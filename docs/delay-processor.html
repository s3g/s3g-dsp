Delay Processor

The s3g Delay Processor is the first stable native effect in this repository. It treats each active channel as a parallel lane with its own delay time, feedback, tone, pitch, smear, and cross-lane diffusion behavior.

The important idea is that these per-lane values are not exposed as hundreds of host parameters. Instead, the plugin exposes a compact topology surface. Moving that surface changes the relationships between lanes.

Workflow

Patch Matrix

Active input rows define the lane count used by the topology. The 8ch processor can be used as 2ch, 4ch, 6ch, 8ch, or a custom matrix.

Point Field

Shape, displacement, animation, and variants position the active lanes as points in a 3D field.

Delay Engine

Each point is converted into smoothed per-lane DSP values while the host automation surface stays compact.

Topology Map

The topology map is a readable way to understand what the Delay Processor is doing internally. The point field is not only a visualization: point position, radius, lane phase, nearest neighbors, and centroid pull are used by the delay generator.

Control Model

  • Patch matrix controls which lanes are active.
  • Topology controls generate per-lane differences.
  • ANIM and VAR provide motion scene behavior using the same topology controls.
  • Mix remains a global dry/wet control.
  • Per-lane internals remain hidden from host automation.
