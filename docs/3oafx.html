3OAFX

3OAFX pages cover ambisonics-specific plugins and infrastructure. These tools are separate from the general multichannel effects and instruments because channel order, spatial format, and host routing are part of the instrument.

The current target is REAPER on macOS, using ACN/SN3D ambisonics and fixed-width buses where predictable routing matters. The point encoder is third-order; the speaker decoder prototype allows first through seventh order.

How This Differs

General effects in Effects process multichannel lanes without assuming an ambisonic format. Instruments in Instruments generate or play sound internally. 3OAFX plugins assume a spatial format or a 3OAFX workflow boundary.

  • Point-source encoders expose per-point AED/gain automation.
  • Speaker decoders expose layout, order, matrix mode, and custom speaker placement.
  • Direct layout panners route sources to speaker layouts without an ambisonic encode/decode stage.
  • Boundary plugins keep fixed bus layouts so REAPER pin routing stays predictable.
  • 24-channel virtual speaker inserts are treated as spatial infrastructure, not generic channel counts.
