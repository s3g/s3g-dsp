Ambisonics

Ambisonics pages cover ambisonics-specific plugins and infrastructure. These tools are separate from the general multichannel effects and instruments because channel order, spatial format, and host routing are part of the instrument.

The supported target is REAPER on macOS, using ACN/SN3D ambisonics and fixed-width buses where predictable routing matters. The point encoder and 3OAFX effects are third-order; decoders and utilities support first through seventh order.

How This Differs

General effects in Effects process multichannel lanes without assuming an ambisonic format. Instruments in Instruments generate or play sound internally. Ambisonics plugins assume a spatial format or a 3OAFX virtual speaker workflow.

  • Point-source encoders expose per-point AED/gain automation.
  • Speaker decoders expose layout, order, matrix mode, and custom speaker placement.
  • Direct layout panners route sources to speaker layouts without an ambisonic encode/decode stage.
  • Ambi Group processors treat each 16-channel 3OA feed as a locked bus group.
  • 3OAFX effects use a one-effect-per-plugin model so masking and gain staging stay predictable.
  • The 24-point virtual speaker layer is internal to the 3OAFX effects.

3OAFX Processing Flow

The current CLAP 3OAFX path uses ordinary third-order ACN/SN3D plugin inserts. Each effect handles the 24-point virtual speaker layer internally, then returns a 3OA output.

3OA source
-> s3g 3OAFX Delay, Filter, Gain, or Pitch
-> optional additional 3OAFX effects
-> Ambi Speaker Decoder, Ambi Stereo Decoder, or Ambi Head Decoder

Inside each 3OAFX effect, the signal path is:

3OA input
-> decode to 24 virtual speaker points
-> dry copy path
-> one effect return and one AED return mask
-> mixer
-> encode back to 3OA output

The dry copy stays separate from the effect return until the mixer. Stack separate plugin instances in series when the order matters. For parallel combinations, use REAPER routing rather than an internal rack mixer.
