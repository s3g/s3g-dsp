.s3gray Format

.s3gray is the runtime interchange format between s3g-mc Ray Sketch and s3g Ambi Ray Encoder. It is UTF-8 JSON with a dedicated extension, separate from the editable Ray Sketch project. It contains resolved directional events rather than audio or a realtime geometry engine.

Version 1

The root object contains:

  • format: s3g-ambi-ray-field
  • version: 1
  • ambisonics: ACN/SN3D metadata
  • interpolation: sampled-cell and stable-slot interpretation metadata
  • space: optional generalized geometry, regions, portals, and display provenance
  • room: dimensions, top-view polygon, ceiling profile, navigation bounds, and material summary
  • listener_position_m and default_source_position_m: XYZ positions in metres
  • grid: authoring-grid dimensions, source-sampling metadata, and the optional global stable-slot key list
  • cells: one to 256 sampled source positions

Each cell has a position_m, up to 32 early_reflections, and one late profile. Every early reflection carries a stable slot, delay in milliseconds, gain, azimuth, elevation, and normalized damping. It may also carry bounce_position_m, an optional XYZ wall, floor, ceiling, portal, or echo-path point used for path display. A slot may be absent from a cell when that path is not active there; readers treat its gain as zero. The late profile carries start time, decay time, level, diffusion, and damping.

Runtime Interpretation

The source XYZ parameters map to the field's navigation bounds. Ambi Ray Encoder computes source-to-listener distance, delay, attenuation, and direction continuously for the direct sound. It blends stable reflection slots and late profiles from the four nearest cells using inverse-distance weights. Position and field changes are smoothed by the plugin's MOVEMENT control.

The direct component follows the selected first-through-seventh ambisonic order. Early and late room components are represented through third order. Timing is sample-rate independent in the file and compiled for the current host rate. Imported JSON is embedded in CLAP project state.

Compatibility

The editable Ray Sketch project uses the s3g-ray-sketch format identifier and is not a plugin input. Ray Sketch may import Imprint Sketch and legacy room-sketch projects as authoring sources, but Ambi Ray Encoder loads only a version 1 s3g-ambi-ray-field runtime file.

Geometry, materials, path losses, bounce points, and decay values are creative acoustic estimates intended for repeatable spatial composition. Readers derive sound from the resolved delay, gain, and AED values; bounce_position_m and generalized geometry are optional display provenance, not a requirement to ray trace on the audio thread. Version 1 readers may ignore the bounce field.
