Ambi Cloud Encoder

s3g Ambi Cloud Encoder 64 distributes up to 64 input lanes into one to four spatial clouds, then encodes the result as first-through-seventh-order ACN/SN3D.

Use it when a group of mono or direct-channel sources should behave like a moving spatial mass rather than individually placed point sources. For explicit per-point automation, use Ambi Point Encoder.

Model

  • INPUTS sets how many input lanes feed the encoder.
  • CLOUDS sets the number of cloud centers, from one to four.
  • Input lanes are assigned across the active clouds and spread around each cloud center.
  • ORDER selects the output ambisonic order from 1OA to 7OA.
  • The output bus is fixed at 64 channels so REAPER routing stays stable as order changes.

Cloud Editing

The selected cloud has azimuth, elevation, distance, and gain controls. Each cloud center is also exposed to REAPER automation as its own azimuth, elevation, and distance lane.

  • AZ and EL place the selected cloud center.
  • DIST changes cloud distance and affects both display radius and encoded level.
  • GAIN trims the selected cloud before encoding.
  • OUT trims the final encoded output.

Shape and Motion

SHAPE changes how sources are distributed around each cloud. Current shapes include CUMULUS, STRATUS, CIRRUS, LENTICULAR, and STORM.

FORCE changes the internal motion behavior. CALM stays slow and centered. ADVECTION, SHEAR, CONVECTION, and TURBULENCE add stronger cloud movement.

  • SPRD widens the azimuth spread around each cloud.
  • E-SPR widens the elevation spread.
  • JIT adds source-position irregularity.
  • DRFT adds continuous source drift.
  • RATE sets motion speed.
  • DECOR applies a small decorrelation trim between lanes.

Routing

Place the plugin on a wide REAPER track carrying source lanes. Decode the output with Ambi Speaker Decoder, Ambi Stereo Decoder, or Ambi Head Decoder.
