Ambi Head Decoder

s3g Ambi Head Decoder decodes ACN/SN3D ambisonic input to stereo using a code-generated synthetic head model. It can run as a headphone-style binaural monitor or add a transaural crosstalk-cancellation stage for stereo loudspeakers.

Model

The first version does not embed measured HRTF datasets and does not require external SOFA files. The decoder uses short interaural delays, head-shadow filtering, and simple pinna/width controls to create a lightweight synthetic head response.

DIRECT mode maps ambisonics to an internal spherical head grid without exposing a named speaker layout. Virtual field mode samples ambisonics through a selected virtual field before the head model. The field path is useful for matching the visual language of the speaker decoder; it is not required for binaural decoding.

This keeps the pre-release build simple: no external HRTF file paths, no dataset redistribution, and no extra license files. Measured HRTF or SOFA loading can be added later as a separate advanced layer.

Head Profiles

  • Medium head: default synthetic profile.
  • Small head and Large head: change the broad shadow/pinna profile without implying a measured listener.
  • EAR: independent ear-width control in centimeters. This affects interaural delay and the GUI head/ear spacing separately from the selected head profile.

Modes

Binaural is intended for headphones. Transaural starts with the same stereo ear signal, then applies a crosstalk-cancellation stage for loudspeaker playback. Transaural mode depends strongly on speaker angle, listener position, and room acoustics.

The transaural stage is based on the same practical model as the s3g-mc JSFX crosstalk canceller: delayed and filtered opposite-channel cancellation, optional matrix-inverse-style compensation, low-frequency protection, stereo preserve, and output safety gain.

Controls

  • ORD: ambisonic order, from 1OA to 7OA.
  • DEC: decoder path. DIRECT uses an internal spherical head grid; Virtual field uses the selected field layout.
  • FIELD: virtual field layout, active when DEC is set to Virtual field.
  • WGT and AGN: ambisonic order weighting and output autogain.
  • HEAD: synthetic head profile in the BINAURAL toolbox.
  • MODE: binaural or transaural.
  • YAW and PIT: listening direction.
  • WID, PIN, and EAR: stereo width, pinna influence, and ear-width spacing for the synthetic head model.
  • ROOM, XTC, XMOD, SPK, LOW, and KEEP: room/diffuse blend, transaural cancellation amount, mode, speaker half-angle, low-frequency protection, and stereo preserve.
  • OUT: final output gain.

REAPER Routing

This plugin exposes a true stereo output bus. On wide REAPER tracks, higher track channels may still pass around a true stereo CLAP plugin unless the host pin connector clears them. If a following meter shows channels above 2, open the plugin pin connector and enable Zero out unmapped output channels.

This appears to be a host pin-routing behavior with true stereo CLAP outputs on wider REAPER tracks. Fixed-width s3g-dsp plugins can clear their own unused lanes because those lanes remain part of the plugin output bus. A true stereo decoder cannot reliably clear channels that REAPER routes around the plugin.

Use Ambi Stereo Decoder for loudspeaker-style stereo pickup models. Use this Head Decoder when the monitoring target is headphones or a transaural loudspeaker setup.
