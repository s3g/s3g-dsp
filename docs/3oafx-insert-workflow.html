3OAFX Processing Workflow

The current CLAP 3OAFX path uses ordinary third-order ACN/SN3D plugin inserts. Each 3OAFX effect handles the 24-point virtual speaker layer internally, then returns a 3OA output.

For the current effect set, see 3OAFX Effects. For point-source encoding and AED automation, see Ambi Point Encoder. For speaker playback and virtual layout testing, see Ambi Speaker Decoder. For direct source-to-speaker routing, see Layout Panner.

Signal Path

3OA source
-> s3g 3OAFX Delay / Pitch / Filter / Gain
-> optional additional 3OAFX effects
-> Ambi Speaker Decoder, Ambi Stereo Decoder, or Ambi Head Decoder

Inside each 3OAFX effect, the signal path is:

3OA input
-> decode to 24 virtual speaker points
-> dry copy path
-> one effect return and one AED return mask
-> mixer
-> encode back to 3OA output

The dry copy stays separate from the effect return until the mixer. This follows the useful part of the JSFX send/return model without exposing a 72-channel bus in REAPER.

Stacking Effects

Use separate plugin instances for separate effects. This keeps gain staging and mask behavior readable: one delay mask, one pitch mask, one filter mask, or one gain/cut mask per plugin.

Stack plugins in series on the same 3OA track when the order matters. For parallel combinations, use REAPER routing rather than an internal rack mixer.

Plugin Shape

The current 3OAFX effects expose 16-channel 3OA input and output buses. Put them on a 16-channel 3OA track before a decoder or visualizer.

The older rack and boundary-pair experiments remain source checkpoints only. They are not part of current CLAP builds.
